Tactile Dexterity: Manipulation Primitives with Tactile Feedback

Francois R. Hogan, Jose Ballester, Siyuan Dong, Alberto Rodriguez

Summary [video] [paper]

  • Closed loop tactile controller for dual palm
  • Contact state and object state control (one compensates for the other)
  • Decompose plans into a set of primitives
  • Key question: How do we structure manipulation planning to use tactile feedback?

Introduction

  • Critique on vision-based sensing:
    • Tough for accurate, dextrous manipulation
    • Eg: object re-orientation, insertion
  • Motivation for tactile sensing:
    • Can localize contact geometry
    • Track contact motion
    • Infer contact forces
  • Tactile dexterity approach:
    • Focuses on contact interactions with rich geometric information for object state estimation
    • Four manipulation primitives: grasp, push, pivot, and pull
    • State control is divided into controlling contact state and object state
  • Tested with GelSlim moving object from start to target pose

Object state control

  • Tactile controller fights external perturbations but object trajectory is disturbed
  • Two kinds of tracked features:
    • points (corners)
    • lines (edges)
  • Tactile state estimation is formulated as an optimization problem:
    • Update object pose p0p_0p0​﻿ such that the two geometric constraints are satisfied

    • Enforce known model to align detected lines with model, and detected edges with points
